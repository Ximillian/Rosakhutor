# Детектор аномальных паттернов для оператора СКД (демо-версия)

Система поддержки принятия решений для службы безопасности горнолыжного курорта. В реальном времени анализирует события проходов через турникеты, выявляет подозрительные однодневные ски-пассы с помощью модели машинного обучения и предоставляет оператору ранжированный список для проверки.


## Возможности

\- Загрузка предобученной модели (joblib) и «горячая» замена при обновлении файла.

\- Имитация потока событий из CSV-файла (демо-режим) с возможностью замены на реальный источник (ClickHouse, Kafka).

\- Вычисление поведенческих признаков на лету (скользящее окно 30 минут, агрегация за день).

\- Инференс модели LightGBM с PU-взвешиванием.

\- Ранжирование пассов по максимальной вероятности перепродажи.

\- Веб-интерфейс оператора с автоматическим обновлением топ‑N, аутентификацией и возможностью блокировки.

\- Логирование действий оператора для накопления обратной связи.

\- Планировщик периодических задач: сброс дня, сохранение состояния, проверка обновления модели.

## Конфигурация

Параметры задаются в config.yaml. Пример:

model\_path: "weighted\_model.pkl"

data\_csv: "March.csv"

window\_minutes: 30

early\_profile\_minutes: 90

lunch\_start: 13

lunch\_end: 15

top\_n: 20

state\_csv: "state\_store.csv"

audit\_csv: "audit\_log.csv"

web\_host: "0.0.0.0"

web\_port: 5000

web\_debug: false

refresh\_interval: 60



model\_path – путь к файлу модели (joblib).

data\_csv – путь к CSV с данными событий (для демо).

window\_minutes – размер скользящего окна для признаков.

early\_profile\_minutes – длительность раннего профиля (первые N минут).

lunch\_start / lunch\_end – часы обеденного периода.

top\_n – количество подозрительных пассов в веб-интерфейсе.

state\_csv – файл для сохранения состояния (между перезапусками).

audit\_csv – файл журнала действий оператора.

web\_host / web\_port – адрес и порт веб-сервера.

web\_debug – включить отладочный режим Flask.


## Запуск

1. Убедитесь, что файлы weighted\_model.pkl, March.csv, KD-trass.csv и config.yaml находятся в корне проекта.

2. Создайте и активируйте виртуальное окружение (опционально).

3. Установите зависимости: pip install -r requirements.txt

4. Запустите приложение:
      python main.py

5. Откройте браузер и перейдите по адресу http://127.0.0.1:5000.

6. Введите логин: operator, пароль: password.

7. Наблюдайте обновляемую таблицу с подозрительными ски-пассами.


## Как работает

При старте из CSV загружаются все события (за весь месяц). Они сортируются по времени и «проигрываются» с небольшой задержкой, имитируя реальный поток.

Для каждого нового подъёма вычисляется срез из 20 признаков, отражающих темп, вариативность, энтропию турникетов, биграммы переходов и др.

Модель LightGBM выдаёт вероятность перепродажи для данного среза.

Для каждого ски-пасса обновляется максимальная за день вероятность.

Пассы ранжируются по убыванию максимума, и топ‑N отображается в веб-интерфейсе.

При смене даты (по данным CSV) состояние автоматически сбрасывается – начинается новый день.

Оператор может заблокировать подозрительный пасс; действие записывается в журнал аудита.

Состояние (накопленные максимумы) периодически сохраняется в state\_store.csv и восстанавливается при перезапуске, если день ещё не завершён.
